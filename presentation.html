Motivation

Problem Statement

Traditional structural inspection methods are costly, dangerous, and slow.

⚠

Safety Hazards

Manual inspections require workers at dangerous heights, risking falls and injuries on scaffolding and ladders.

⏱

Time-Intensive

Full building inspections can take days or weeks, causing operational delays and increased labor costs.

👁

Human Error

Visual assessments are subjective and inconsistent. Small cracks and early-stage defects are often missed.

Our Goal: Automate the entire inspection workflow — from flight to defect detection to reporting — using an autonomous drone paired with AI vision analysis.

Our Approach

Proposed Solution

A three-stage autonomous pipeline: Fly, Analyze, Report.

✈

1. Autonomous Flight

A Mavic 2 Pro drone autonomously flies a rectangular path around a building, capturing 16 high-resolution images (4 per side).

🧠

2. AI Defect Analysis

Each image is analyzed by Groq's Llama 4 Scout vision model for crack detection, surface deterioration, and structural risk assessment.

📊

3. Real-Time Dashboard

Results stream to a live web dashboard via Server-Sent Events, showing defect details, risk levels, and maintenance recommendations.

Component 1

Autonomous Drone Controller

Finite State Machine

The drone follows a deterministic state machine:

  1. TAKEOFF — Ascend to half the building height
  2. STABILIZE — Hover for 3 seconds (PID settling)
  3. SIDE 1–4 — Strafe along each wall, capture 4 photos
  4. TURN 1–3 — Yaw left 90° at each corner
  5. LAND — Descend and shut down motors

PID Flight Control

Adapted from the official Webots Mavic 2 Pro controller:

  • Roll PID — Lateral stability (Kp = 50.0)
  • Pitch PID — Forward/backward control (Kp = 30.0)
  • Vertical PID — Altitude hold (Kp = 3.0)
  • Yaw control — Heading correction during strafe
  • Forward-drift correction — GPS-based compensation

Key Innovation: Distance-based image capture using GPS decomposition into forward/lateral components relative to the drone's heading — ensuring evenly spaced photos regardless of flight speed variations.

Navigation

Flight Path & Image Capture

Capture Strategy

  • 4 images per side at 25%, 50%, 75%, 100% of wall length
  • 16 total images for full 360° coverage
  • Camera tilted 17° downward for optimal wall coverage
  • GPS-based distance tracking triggers captures at precise locations
  • Flight altitude: half the building height (configurable)
  • Building dimensions loaded from config.json

The drone decomposes its GPS displacement into forward and lateral components to measure true strafe distance, compensating for any drift.

Component 2

AI-Powered Defect Analysis

Analysis Pipeline

  1. File Watcher monitors inspection_images/ directory for new captures
  2. Base64 encoding of images for API transmission
  3. Sent to Groq Cloud API running Llama 4 Scout vision model
  4. Structured JSON response with defect analysis
  5. Results POSTed to Flask dashboard for display

Detection Capabilities

🔍 Crack Detection

Type, severity, direction, and length classification

📈 Surface Assessment

⚠️ Risk Classification

Low / Medium / High / Critical severity ratings

Recommended repair and follow-up procedures

Why Groq? Groq's LPU inference engine provides ultra-fast processing (~2–5 seconds per image). The free tier supports 30 requests/min and 1,000 requests/day — more than sufficient for building inspections.

Technical Depth

Key Implementation Details

🛠

PID Motor Mixing

Four-motor velocity mixing adapted from the official Webots C controller. Each motor receives a combination of vertical thrust, roll input, pitch input, and yaw input to achieve stable flight.

fl = K_THRUST + vert - roll + pitch - yaw fr = K_THRUST + vert + roll + pitch + yaw rl = K_THRUST + vert - roll - pitch + yaw rr = K_THRUST + vert + roll - pitch - yaw
📍

GPS Displacement Decomposition

Raw GPS deltas are decomposed into forward and lateral components relative to the drone's current heading. This ensures accurate distance-based triggers even under wind or drift.

forward = dx*cos(h) + dy*sin(h) lateral = dx*sin(h) - dy*cos(h)
📷

Gimbal Stabilization

Camera roll and pitch motors are servo-controlled using gyroscope feedback. A fixed 0.3 rad (17°) downward tilt ensures the camera faces the building wall for optimal image quality.

🔄

Configurable Dimensions

Building length, breadth, and height are stored in config.json. The flight path, capture points, and altitude are all dynamically computed from these three values.

Running the System

Three concurrent terminals power the full pipeline.

▶

Terminal 1: Simulation

webots worlds/inspection_world.wbt

Opens the Webots world. Press Play to launch the drone controller.

▶

Terminal 2: Dashboard

cd dashboard python app.py

Starts the Flask server on localhost:5000.

▶

Terminal 3: Analyzer

cd analysis python analyzer.py

Watches for new images and sends each to Groq for analysis.

Prerequisites: Webots R2025a, Python 3.10+, Groq API key (free). Install dependencies with pip install -r requirements.txt.

Reflection

Challenges & Solutions

🔄

Drone Oscillation on Takeoff

Problem: Aggressive PID gains caused wild oscillation after takeoff.

Solution: Introduced a 3-second STABILIZE phase to allow PID transients to settle before beginning the inspection path.

📋

Forward Drift During Strafe

Problem: The drone drifted toward the building while strafing sideways.

Solution: GPS-based forward-drift correction applies a compensating pitch disturbance proportional to the measured drift.

📷

Inaccurate Image Spacing

Problem: Time-based capture produced unevenly spaced images due to variable drone speed.

Solution: Switched to distance-based triggering with GPS displacement decomposition for precise spatial intervals.

🧠

API Rate Limiting

Problem: Rapid image captures could exceed Groq's 30 req/min free tier limit.

Solution: Built-in retry logic with exponential backoff handles transient 429 responses gracefully.

Historical Comparison

Store inspection results in a database and compare across time to track defect progression and predict maintenance needs.

Multi-Drone Coordination

Extend the system to support fleet operations with multiple drones inspecting different buildings or building faces simultaneously.

Real Hardware Deployment

Port the controller to a physical DJI Mavic drone using the DJI SDK, transitioning from simulation to real-world inspections.
